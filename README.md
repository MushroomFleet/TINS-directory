# TINS Directory

A curated collection of **TINS (There Is No Source)** implementation plans â€” complete, ready-to-build game design documents created using the TINS methodology.

Each plan in this repository is a self-contained implementation blueprint. Hand one to an AI coding assistant with the prompt:

> *"We can begin completion of the `TINS-NAME.md` implementation plan."*

...and watch a full game materialize from a single document.

## What is TINS?

**There Is No Source** is a documentation-first approach to game development where the entire game â€” mechanics, architecture, visual design, audio, state management â€” is specified in a single comprehensive README before any code is written. The resulting document serves as both a design bible and an implementation contract.

Learn more at [thereisnosource.com](https://www.thereisnosource.com)

## How to Use These Plans

1. **Pick a game** from the `/GAMES/` folder
2. **Open the markdown file** and review the implementation plan
3. **Start a new project** and paste the plan to your AI coding assistant
4. **Prompt:** `"We can begin completion of the [FILENAME] implementation plan."`
5. **Build** â€” the plan contains everything needed for a complete implementation
